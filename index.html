Welcome to Angulpar seed.

You can use this project as a boilerplate removal for an full JS Stack Architechture, includes friendly CI-CD Travis integration and Firebase login is enabled and defined in Swagger as an example

What for?

The main idea is to have a Angular/NodeJS project with a Swagger defined API to connect with and Elastic Search back-end, all the CI-CD process is automated and working out-of-the-box (when finished)

Current Status

I finished migration to Gulp, I am connect the Swagger Client code generation in Gulp ATM, I will move the Firebase login logic from the AngularJS to the NodeJS to have real security, after that everything will be automated and only changing 1 field in Swagger API will generate a new client so we just define the API and use it for doing things, not need to implement the call, with the promise etc etc, the client is Angular auto-generated code
